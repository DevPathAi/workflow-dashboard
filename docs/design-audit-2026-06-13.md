## 첫인상 (Dashboard)

- 사이트가 **"진지하고 절제된 사내 데이터 도구"**를 전달한다. 차분한 그레이스케일, 군더더기 없음.
- 가장 먼저 눈이 가는 3가지: (1) 우상단 큰 검은 **0%**, (2) "트랙별 현황"의 카드 그리드, (3) 하단 비어 보이는 추이 차트. — 의도한 위계라면 좋지만, 정작 **모든 카드가 똑같이 0%**라 어디를 봐야 할지 진입점이 없다.
- 한 단어로: **"비어 있다(empty)".**
- **Page Area Test**: "트랙별 현황", "주차별 상세", "진행률 추이" 영역은 2초 안에 목적을 말할 수 있다(합격). 단, 각 트랙 카드 하단의 W1~W24 스트립은 무엇인지 즉시 알기 어렵다(미니 타임라인인지 노이즈인지).

## 추출된 디자인 시스템 (렌더 기준)

- **폰트**: `ui-sans-serif, system-ui, sans-serif` 단일 — 커스텀 서체 없음(Tailwind 기본).
- **컬러**: 거의 순수 그레이스케일(oklch 0.985 / 0.923 / 0.869 / 0.709 / 0.444 + black/white). UI 크롬에 **액센트/브랜드 컬러 0개**. (차트 범례 색은 canvas 렌더라 별개)
- **헤딩 스케일**: H1 20px/700, H2·H3 모두 14px/600 → 사실상 **20px·14px 2단계**. 섹션 제목이 본문보다 거의 안 큼.
- **본문**: 16px ✓ · **최소 텍스트**: **7px**(트랙 카드 주차 라벨) · 카테고리 라벨 10px.

## 발견 사항 (영향도 순)

### 🔴 HIGH

**FINDING-001 — 트랙 카드 주차 라벨 7px, 모든 폭에서 판독 불가** · Typography
9개 카드 각각에 W1~W24 미니 스트립이 **7px**로 렌더된다. 데스크톱에서도 라벨이 충돌하고("W20W21", "W22W23W24"), 모바일에서는 알아볼 수 없는 얼룩이 된다. 캡션 최소 12px 기준을 크게 밑돈다. 메인 화면에 **9번 반복**되어 가장 손상이 큰 요소. (JS 검증: 최소 font-size = 7px)

**FINDING-002 — 핵심 지표(카드 30px "0%")가 대비 ~2.2:1** · Color & Contrast
각 카드의 큰 "0%"가 `oklch(0.709)` 회색 / 흰 카드 배경 → 대비 약 **2.2:1**로 대형 텍스트 기준 3:1도 미달. 대시보드에서 가장 중요한 숫자가 가장 안 읽힌다. (헤더의 36px "0%"는 검정이라 양호 — 카드 숫자만 해당)

**FINDING-003 — all-zero 상태가 "데이터 로드 실패"처럼 보임** · Content / Hierarchy
모든 카드 0% + 빈 회색 진행바 + 추이 차트가 완전히 평평(라인 없음). "아직 시작 전" 같은 빈 상태 안내가 없어, 처음 보는 사람은 *계획 단계*인지 *고장*인지 구분 불가. (현재 시드는 전부 0% 계획 상태)

**FINDING-004 — 모바일 헤더 충돌** · Responsive
375px에서 "DevPath AI / Workflow Dashboard" 타이틀이 "Settings" 링크 + "0% 전체 진행률" 블록과 겹치고 클리핑됨. 헤더가 리플로우되지 않는다. (mobile 스크린샷)

**FINDING-005 — 주차별 상세 테이블 모바일 미대응** · Responsive
트랙 + W1~W24 + 합계 = **26열** 테이블이 375px에 그대로 눌려 들어가 셀이 판독 불가/가로 넘침. 가로 스크롤 어포던스나 카드형 대체가 없다.

### 🟡 MEDIUM

**FINDING-006 — 타이포 시스템 부재(기본 시스템 폰트)** · Typography / AI Slop
전체가 `system-ui` 기본 스택. 커스텀 서체 없음 = "타이포를 포기한" 신호(슬롭 블랙리스트 #11). 스케일도 20/14 2단계뿐.

**FINDING-007 — 대시보드 위계가 평평(9개 동일 카드)** · Visual Hierarchy
트랙별 색/그룹/강조가 없어 9개 카드가 완전히 동일하게 보인다. 눈의 진입점이 없음.

**FINDING-008 — 카테고리 라벨 10px + 저대비(이중 실패)** · Typography / Contrast
"Backend"/"AI" 등 라벨이 10px(캡션 최소 12px 미달) + `oklch(0.709)` 회색(~2.2:1).

**FINDING-009 — Data Edit 탭 컨트롤 저대비** · Color & Contrast
주차 선택 버튼(W2~W5 비활성)과 "+ Step 추가" 버튼이 흐린 회색이라 거의 유령처럼 보임. (좌표 검증: 요소 겹침은 아님 — 순수 대비/어포던스 문제)

**FINDING-010 — Import/Export "config.json 다운로드" 버튼 거의 안 보임** · Color & Contrast
"설정 내보내기" 섹션의 다운로드 버튼이 흰 배경에 흰색에 가까워 사실상 비가시. 형제인 "데이터 내보내기" 칩들은 테두리가 있어 보이는데 일관성도 깨짐.

### 🟢 POLISH

- **FINDING-011** — 상세 페이지 주차 탭 스트립이 헤더 배경을 넘침(W24가 다크 헤더 밖 흰 영역에 위치). · Layout
- **FINDING-012** — 잘못된 repo URL의 Not-found가 헤더·뒤로가기 없는 맨 텍스트("레포를 찾을 수 없습니다")로 사용자를 고립시킴. · Content
- **FINDING-013** — 상세 Task/Workflow 카드가 좌측 제목·우측 0%만 있고 가운데가 텅 빔(진행바/설명 없음). · Layout
- **FINDING-014** — 파괴적 액션(레포 "삭제", "전체 초기화")이 중립 버튼과 동일 스타일 — 빨강/경고 없음. · Interaction
- **FINDING-015** — Import/Export 섹션 이모지 장식(💾📦📥🗑) — 약한 AI 슬롭 패턴. · AI Slop
- **FINDING-016** — 헤더 "Settings" 링크 터치 타깃 75×20(높이 20px, 44px 미달). · Interaction(mobile)
